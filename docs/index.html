podman-minimal

Run clean Podman dev/runtime environments from any folder with one command. Normal runs are rootless: your tools live in containers, not on the host.

Install in one line

Linux / macOS

python3 -c "import urllib.request as u; exec(u.urlopen('https://raw.githubusercontent.com/vincenzoml/podman-minimal/main/podman-minimal.py').read().decode())" --install

Windows (PowerShell)

python -c "import urllib.request as u; exec(u.urlopen('https://raw.githubusercontent.com/vincenzoml/podman-minimal/main/podman-minimal.py').read().decode())" --install

Install under ~/.local/bin if you want to avoid privilege prompts — see the README on GitHub.

Everyday commands

podman-minimal
podman-minimal nvidia-smi
podman-minimal -v -- python3 -c "print('verbose + image info')"
podman-minimal --nohup run.log -- python3 -m http.server 8080
podman-minimal --dockerfile /path/to/Dockerfile
podman-minimal --help

Behavior

  • Quiet by default — only Podman and your command speak unless you pass -v / --verbose.
  • --nohup — detached run, log file, live copy to the terminal while it stays open.
  • Sudo — not used for normal runs. Used only when installing the binary to a protected path, auto-installing Podman on Linux, optional daemon linger, or optional system Quadlet. Set PODMAN_MINIMAL_NO_SUDO=1 to forbid sudo from this script entirely.
  • Linux — package-manager Podman install. macOS — Homebrew + brew install podman. Windows — winget or choco.
  • Daemon / Quadlet — advanced flows documented in the README (Linux).

Use cases

  • Interactive shell without polluting the host.
  • One-shot commands and CI-like local runs.
  • Per-repo workflow with podman-minimal.py in the tree.
  • Long-running jobs via --nohup or user daemon on Linux.
